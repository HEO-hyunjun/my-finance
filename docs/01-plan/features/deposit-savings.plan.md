## 1. 기능 개요

기존 자산 관리(asset-management)를 확장하여 **예금(deposit)**, **적금(savings)**, **CMA/파킹통장(parking)** 자산 유형을 추가한다. 주식/금과 달리 시세 조회가 아닌 **사용자가 입력한 이율**을 기반으로 이자를 자동 계산하여 자산 가치에 반영한다.

### 1.1 핵심 목표

- 3가지 자산 유형 추가: `deposit` (예금), `savings` (적금), `parking` (CMA/파킹통장)
- 사용자가 연이율을 직접 입력
- 경과 기간에 따른 이자 자동 계산 (세전/세후)
- 만기일 관리 (예금/적금) 및 만기 도래 표시
- 기존 자산 요약에 예금/적금/파킹통장 자산 통합 표시

### 1.2 자산 유형별 차이

| 구분 | 예금 (Deposit) | 적금 (Savings) | CMA/파킹통장 (Parking) |
|------|---------------|---------------|----------------------|
| 납입 방식 | 목돈 일시 예치 | 매월 정기 납입 | **수시 입출금** |
| 만기 | 있음 (고정) | 있음 (고정) | **없음** |
| 이자 발생 | 만기 시 정산 | 만기 시 정산 | **매일 발생** |
| 잔액 변동 | 없음 | 월 1회 증가 | **수시 변동** |
| 이자 계산 | 원금 x 이율 x (경과일/365) | 월납입액 x 이율 x n(n+1)/24 | **일일잔액 x 이율/365** |
| 주요 입력 | 원금, 연이율, 만기일 | 월납입액, 연이율, 만기일 | **현재잔액, 연이율** |
| 현재 가치 | 원금 + 경과이자 | 누적납입액 + 경과이자 | **잔액 + 일일이자** |

## 2. 구현 범위

- [ ] **Asset 모델 확장**: 예금/적금/파킹 전용 필드 추가 (interest_rate, maturity_date, start_date, monthly_amount, principal, tax_rate, bank_name)

  - 예금: 단리/복리 선택 가능
  - 적금: 정액적립식 단리 이자 계산
  - 파킹통장: 현재잔액 기반 일일이자 계산
  - 세후이자: 이자소득세 15.4% 자동 적용
- [ ] **Asset Summary 확장**: 예금/적금/파킹 자산을 원금+경과이자로 평가
- [ ] **Alembic 마이그레이션**: Asset 테이블 필드 추가

#### Frontend
- [ ] **타입 확장**: AssetType에 deposit, savings, parking 추가 + 관련 필드
- [ ] **자산 추가 모달 확장**: 예금/적금/파킹 선택 시 유형별 전용 입력 폼
- [ ] **AssetCard 확장**: 이율, 만기일(해당 시), 경과이자 표시
- [ ] **자산 요약 카드**: breakdown에 예금/적금/파킹 카테고리 추가
- [ ] **이자 계산 표시**: 세전이자, 세후이자, 만기 예상 금액(해당 시) 표시

### 2.2 Out of Scope

- 실제 은행 API 연동 (오픈뱅킹)
- 만기 도래 푸시 알림 (알림 기능은 별도 feature)
- 복리 적금 (대부분의 한국 적금은 단리)
- 자유적금 (변동 납입액) — 정액적립식만 지원
- 예금/적금 자동 해지/갱신
- 파킹통장 과거 구간별 정밀 이자 계산 (입출금 이력 기반) — 현재 잔액 기준 계산으로 간소화

## 3. 기술 설계 방향

### 3.1 데이터 모델 확장

**Asset 테이블 추가 필드** (nullable — 기존 자산에 영향 없음):

| 필드 | 타입 | 설명 | deposit | savings | parking |
|------|------|------|:-------:|:-------:|:-------:|
| `interest_rate` | DECIMAL(5,3) | 연이율 (%, 예: 3.500) | O | O | O |
| `interest_type` | ENUM | 단리(simple) / 복리(compound) | O | - | - |
| `start_date` | DATE | 가입일 | O | O | - |
| `maturity_date` | DATE | 만기일 | O | O | - |
| `monthly_amount` | DECIMAL(18,0) | 월 납입액 (원) | - | O | - |
| `principal` | DECIMAL(18,0) | 원금/현재잔액 (원) | O | - | O |
| `tax_rate` | DECIMAL(5,3) | 이자소득세율 (기본 15.4%) | O | O | O |
| `bank_name` | VARCHAR(50) | 은행명/증권사명 | O | O | O |

### 3.2 이자 계산 로직

```
[예금 — 단리]
세전이자 = 원금 × (연이율/100) × (경과일수/365)
세후이자 = 세전이자 × (1 - 세율)
현재가치 = 원금 + 세후이자

[예금 — 복리 (월복리)]
세전이자 = 원금 × (1 + 연이율/100/12)^경과월수 - 원금
세후이자 = 세전이자 × (1 - 세율)
현재가치 = 원금 + 세후이자

[적금 — 정액적립식 단리]
총납입액 = 월납입액 × 납입회차
세전이자 = 월납입액 × (연이율/100/12) × n(n+1)/2
  (n = 납입한 개월 수, 각 회차별 이자 합산)
세후이자 = 세전이자 × (1 - 세율)
현재가치 = 총납입액 + 세후이자

[CMA/파킹통장 — 일일이자]
일일이자 = 현재잔액 × (연이율/100) / 365
월예상이자 = 일일이자 × 30
연예상이자 = 현재잔액 × (연이율/100)
세후 월예상이자 = 월예상이자 × (1 - 세율)
현재가치 = 잔액 (이자는 별도 표시, 자동 재투자 가정하지 않음)

※ 파킹통장 간소화: 잔액 변동 이력이 아닌 "현재 잔액" 기준으로 일일이자 표시.
  사용자가 잔액을 수정하면 이자 계산도 즉시 반영.

[만기 예상 금액] (예금/적금만)
만기시 총 이자 = 위 공식에서 n = 총 개월수로 계산
만기 수령액 = 원금(또는 총납입액) + 세후이자
```

### 3.3 기존 계산 로직과의 통합

| 자산 유형 | 현재가 결정 방식 | asset_service 변경 |
|-----------|-----------------|-------------------|
| stock_kr, stock_us, gold | MarketService (SerpAPI) | 변경 없음 |
| cash_krw | 1.0 고정 | 변경 없음 |
| cash_usd | 환율 조회 | 변경 없음 |
| **deposit** | **원금 + 경과이자 (계산)** | **신규 분기** |
| **savings** | **누적납입 + 경과이자 (계산)** | **신규 분기** |
| **parking** | **잔액 (이자는 별도 표시)** | **신규 분기** |

## 5. 리스크 및 고려사항

| 리스크 | 영향 | 대응 방안 |
|--------|------|-----------|
| Asset 테이블 필드 과다 증가 | 스키마 복잡성 | nullable 필드로 추가, 필요시 별도 테이블 분리 검토 |
| 이자 계산 정확도 | 실제 은행과 미세한 차이 가능 | 단리 기본, 원단위 절사 적용 (한국 은행 관행) |
| 세율 변경 시 소급 적용 | 기존 자산의 이자 재계산 | 세율은 자산별로 저장, 변경 시 사용자가 수정 |
| 적금 중도해지 처리 | 이자 계산 변경 | 1차 구현에서는 미지원 (Out of Scope) |
| 파킹통장 잔액 동기화 | 실제 잔액과 기록 불일치 가능 | 사용자가 수동 잔액 업데이트, 알림 없음 |
| 파킹통장 이율 변경 | CMA 이율은 수시 변동 가능 | 사용자가 이율 직접 수정, 변경 이력 미관리 |
